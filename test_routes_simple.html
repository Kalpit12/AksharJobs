üìã Test Checklist

Manual testing steps to verify authentication flow:

  1. Initial State (Logged Out):
    • Visit home page - should show Login/Signup buttons in header
    • Profile dropdown should NOT be visible
    • Try accessing protected routes - should redirect to login
  2. Login Process:
    • Go to login page and login with valid credentials
    • Should redirect to appropriate dashboard based on role
    • Header should now show profile dropdown instead of Login/Signup
  3. Authenticated State:
    • Should be able to access protected routes
    • Profile dropdown should show user info
    • Role-specific routes should work based on user role
  4. Logout Process:
    • Click logout from profile dropdown
    • Should redirect to home page
    • Header should immediately show Login/Signup buttons
    • Profile dropdown should disappear
    • Trying to access protected routes should redirect to login
