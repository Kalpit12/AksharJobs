This tool will help fix the 422 errors caused by invalid JWT tokens after switching to MongoDB Atlas.

Click "Check Current Status" to see what tokens are stored

What's Happening?

The 422 errors occur because:

  • Your JWT token was signed with the old JWT secret (local MongoDB)
  • The backend now uses a different JWT secret (Atlas configuration)
  • The token signatures don't match, causing authentication failures

Solution:

  1. Click "Clear All Tokens" to remove invalid tokens
  2. Click "Go to Login Page" to log in fresh
  3. New tokens will be generated with the correct signature
